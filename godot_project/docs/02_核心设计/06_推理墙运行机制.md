# 《血字的研究》推理墙运行机制设计方案
## 文档定位
本文档为推理游戏《血字的研究》核心模块「推理墙」的完整运行机制标准，覆盖线索管理、假设构建、验证判定、矛盾排查、进度管理全流程，用于指导前端开发、策划配置与UI设计工作。机制以真实侦破逻辑为内核，兼顾新手入门门槛与高阶玩家深度体验，采用统一后台逻辑+分层前端交互的设计，保障玩法完整性与工程落地性。

> **MVP / M2+ 功能分层说明**（与《08_系统框架设计》对齐）：
> - ✅ **MVP 必做（核心基础层）**：线索库四态机、三级标签体系、推理看板（拖拽模式）、假设树三级结构、MVP简化验证算法（纯计数两档）、结论里程碑基础点亮
> - 🚧 **M2+ 扩展（扩展进阶层）**：可信度评估体系、矛盾疑点册（含线索对比台+求助全局扫描）、案件进度轴、完整支持度计算公式、自由连线模式、自由猜想功能、被动提示系统、凶手行动线
> - 🎯 **全局支撑层**（贯穿全阶段）：三星评价体系、徽章系统、软性引导系统、存档系统
> - 本设计文档描述完整功能集，实际开发阶段以系统框架分层为准

## 核心设计原则
1.  **逻辑统一，交互分层**：后台共用一套线索匹配、数值计算、矛盾检测逻辑，前端提供多套交互模式适配不同玩家偏好，避免重复开发。
2.  **探索自主，提示可控**：矛盾与推理引导不强制推送，通过难度配置+主动功能入口实现分层，新手可获得充分引导，高阶玩家可享受纯自主探索乐趣。
3.  **渐进解锁，降低门槛**：核心玩法开局即可上手，深度系统随剧情推进逐步解锁，避免初始信息过载。
4.  **标准化配置，落地高效**：采用标签化体系实现内容配置标准化，控制内容生产与调试成本。

## 一、整体系统架构
推理墙采用「核心基础层+扩展进阶层+全局支撑层」的三层架构，核心层常驻承载基础玩法，扩展层随剧情逐步开放，全局层提供通用能力支撑。
```
┌──────────────────────────────────────────────────────────┐
│                     全局支撑层                            │
│     评价体系    难度适配系统    存档系统    数据统计      │
├──────────────────────────────────────────────────────────┤
│                     扩展进阶层（分阶段解锁）              │
│   可信度评估模块   矛盾疑点册   案件进度轴                │
├──────────────────────────────────────────────────────────┤
│                     核心基础层（开局常驻）                │
│                                                          │
│  线索库   ──  推理看板   ──  四级验证体系  ── 结论里程碑  │
│         （拖拽/自由连线双交互模式）                       │
└──────────────────────────────────────────────────────────┘
```

## 二、核心基础层模块
核心基础层为开局即开放的常驻模块，支撑推理玩法的核心循环。

### 2.1 线索库
线索库为全案证据的统一收纳中心，所有现场勘查、证人对话、资料检索获取的信息均在此集中管理。

#### 线索状态机
单条线索全生命周期共4种状态，状态变更同步联动所有关联模块。
| 状态 | 含义 | 触发条件 | UI表现 |
|------|------|----------|--------|
| 未发现 | 线索存在于场景中，玩家尚未交互获取 | 初始默认状态 | 困难模式无提示；普通模式微光标记交互点；简单模式高亮标记交互点 |
| 已收集 | 玩家完成交互后获取完整信息，自动纳入线索库 | 点击场景交互点、完成对话、拿到检验结果 | 进入线索库列表，显示完整描述与对应标签 |
| 已验证 | 通过交叉印证或专业工具检验，确认线索信息可靠 | 双源线索匹配一致、实验室检验结果出具 | 线索卡片边框变为绿色，标注「已验证」标识 |
| 已失效 | 因进度推进或新证据出现，线索失去证明效力 | 超出调查进度窗口、被新证据直接证伪 | 卡片灰色半透明显示，标注「信息失效」标识 |

#### 线索标签体系
所有线索统一配置标准化标签，用于自动匹配假设、计算可信度、识别矛盾，实现内容配置的高效复用。

> **与系统框架字段对应说明**：本标签体系对应 `ClueData` 中的 `content_tags` / `attribute_tags` / `relation_tags` 三字段；而 `discovery_source`（"dialogue"/"observation"/"deduction"/"tool"）是线索**获取方式**分类，两者为不同维度，不冲突。

- **内容标签**：标识线索核心信息，如「车轮印」「身高特征」「毒药」「证词」
- **属性标签**：标识线索来源类型，如「直接物证」「目击证词」「二手传闻」「嫌疑人陈述」，用于可信度计算
- **关联标签**：标识线索所属推理方向，如「马车相关」「凶手特征」「动机相关」，用于自动匹配假设节点

> **说明**：线索的重要性分级（关键/重要/一般/其他/误导）属于**场景设计范畴**，由各案件文档定义，用于判定观察之星的收集完整度；推理墙机制本身不区分线索重要性等级，统一按状态机+标签体系管理。

### 2.2 推理看板
推理看板为玩家进行推理操作的核心区域，支持假设搭建、线索关联、验证提交，提供两套交互模式供玩家自由切换。

#### 假设层级结构
采用「核心问题→主假设→子假设」三级树状结构，保证推理方向清晰，避免无限分支造成混乱。
```
                    核心问题：凶手是谁？作案手法是什么？
                             │
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
    主假设A            主假设B            主假设C
 「马车夫作案」      「入室抢劫伪装」    「熟人仇杀」
          │
    ┌─────┴─────┐
    ▼           ▼
子假设A1     子假设A2
「出租马车」  「凶手返场找戒指」
```

#### 假设状态定义

> **MVP / M2+ 分层**：MVP 阶段实现基础3态（待建立/活跃中/已确认）；领先中、已排除为 M2+ 完整状态机。

| 状态 | 含义 | 转换条件 | 实现阶段 |
|------|------|----------|---------|
| 待建立 | 预设假设节点，尚未关联任何证据 | 初始默认状态，灰色显示 | MVP |
| 活跃中 | 关联至少1条有效证据，具备调查方向 | 关联线索后验证等级达到「证据不足」及以上 | MVP |
| 领先中 | 证据支持度显著高于其他并行假设 | 验证等级达到「倾向成立」，且支持度高于第二名的1.5倍 | M2+ |
| 已确认 | 核心证据链闭合，认定为既定事实 | 验证等级达到「已获证实」，同步至结论里程碑 | MVP |
| 已排除 | 被决定性证据证伪，不再纳入调查范围 | 验证等级处于「矛盾冲突」，且存在决定性反证 | M2+ |

#### 双交互模式
两套模式共用同一套后台计算逻辑，玩家可在设置中随时切换，推理进度完全互通。

##### 模式一：拖拽模式（默认开启）
面向全层级玩家的轻量化操作模式，操作路径短、学习成本低。
- 核心操作：拖拽线索卡片至对应假设节点，自动建立证据关联，实时更新假设验证等级
- 支持操作：线索拖拽至假设建立支持/反对关系、线索拖拽至对比台发起矛盾比对、点击按钮提交假设验证
- 适用场景：快速梳理主线推理，降低操作负担

##### 模式二：自由连线模式（默认关闭，进阶可选 / M2+）

> 🚧 **M2+ 扩展功能**：自由连线模式为高阶玩家的深度操作模式，MVP 阶段仅实现拖拽模式。

面向高阶玩家的深度操作模式，还原真实推理的自由关联过程。
- 核心操作：玩家从任意线索/假设节点拉出连线，自由连接至目标节点，完成关系建立
- 支持的连线类型：
  1.  线索→假设：建立证据支持/反对关系，效果与拖拽模式完全一致
  2.  线索→线索：将两条线索关联比对，等效于拖入对比台，自动触发矛盾检测
  3.  假设→假设：标记两个假设的关联关系（互斥、包含等），用于玩家自主梳理逻辑脉络
- 连线视觉规则：
  - 实线：代表支持关系
  - 红色虚线：代表矛盾/反对关系
  - 灰色虚线：代表待验证/弱关联关系

#### 支持度计算规则（M2+ 完整算法）

> 🚧 **M2+ 扩展**：以下为完整支持度计算公式，MVP 阶段使用纯计数简化算法。

- 单条线索贡献值 = 线索可信度 × 标签匹配系数
- 标签匹配系数由系统根据线索与假设的标签重合度自动判定：
  - 核心标签完全匹配（直接指向假设核心）：×1.0
  - 强相关标签匹配：×0.7
  - 弱相关标签匹配：×0.3
  - 无匹配标签：×0.0
- 假设总支持度 = 所有支持线索贡献值之和 - 所有反对线索贡献值之和

#### 自由猜想功能（M2+）

> 🚧 **M2+ 扩展功能**：MVP 阶段仅支持预设假设节点，不开放玩家自定义猜想。

在预设假设之外，支持玩家自主输入自定义猜想，拖拽任意线索作为支撑证据。系统基于标签匹配度给出定性反馈，不做严格对错判定，对于偏离逻辑的猜想，以角色口吻给出趣味化回复。

### 2.3 四级验证体系

所有假设的验证结果统一划分为4个等级，每个等级对应明确的含义与玩家行动指引。后台保留0-100连续数值计算，前台仅映射为4档等级展示。

> **MVP / M2+ 算法分层**：
> - ✅ **MVP 简化算法**：纯计数法——有反驳证据 → 矛盾冲突；≥3条支持证据 → 已获证实；≥1条支持 → 倾向成立；0条支持 → 证据不足（对应08系统框架实现）
> - 🚧 **M2+ 完整算法**：支持度 = Σ(线索可信度 × 标签匹配系数) − Σ(反对证据 × 可信度)，按百分比映射四级（详见下方支持度计算规则）

| 验证等级 | 系统框架枚举 | 数值区间（完整算法） | 核心含义 | 玩家行动指引 | UI与反馈表现 |
|----------|-------------|---------------------|----------|--------------|--------------|
| 矛盾冲突 | `CONTRADICTORY` | 0-30% | 存在明确反证，假设与证据直接对立，大概率不成立 | 推翻该假设，或寻找证据解释矛盾 | 红色边框+警示图标；明确标注冲突的证据条目 |
| 证据不足 | `INSUFFICIENT` | 31-60% | 有少量线索关联，但支撑力度不足，无法判断方向是否正确 | 补充更多相关证据，或转向其他假设调查 | 黄色边框+提示图标；提示「现有证据不足以支撑该假设」 |
| 倾向成立 | `SUPPORTED` | 61-85% | 多数证据指向该假设，方向大概率正确，但缺少关键闭环证据 | 深挖剩余疑点，寻找决定性证据完成闭环 | 绿色边框+正向图标；提示「证据倾向支持该假设」 |
| 已获证实 | `VERIFIED` | 86-100% | 核心证据链完整闭合，无不可解释的矛盾，可认定为事实 | 假设升级为既定事实，同步点亮结论里程碑 | 金色光效+里程碑动效；触发「推理突破」强反馈 |

> **前台呈现简化**：面向玩家的场景级反馈可简化为三级（正确/存疑/错误），对应关系为：已获证实+倾向成立→正确（绿色对号）、证据不足→存疑（黄色问号）、矛盾冲突→错误（红色叉号）。四级验证为后台完整计算逻辑，三级反馈为前台轻量化呈现，两者不矛盾。

#### 验证触发方式
1.  **主动验证**：玩家点击假设节点的「验证」按钮，随时提交验证并查看报告
2.  **被动提示**：仅满足特定条件时，以角落气泡形式给出弱提示，不强制弹窗：
    - 假设达到「倾向成立」且持续3分钟未验证
    - 新获取的关键证据将假设直接推入「矛盾冲突」
    - 两个主假设均处于「倾向成立」，且支持度差距小于10%

#### 验证报告规范
玩家提交验证后，生成结构化报告，核心包含4部分内容：
1.  假设名称与对应验证等级
2.  支持依据：列出核心支持证据
3.  存疑点：列出冲突证据与核心缺口
4.  行动建议：给出下一步调查方向指引

### 2.4 结论里程碑

结论里程碑为案件侦破进度的可视化载体，以节点点亮形式呈现阶段性成果，给玩家明确的推进成就感。

> **MVP / M2+ 分层**：MVP 实现基础节点点亮逻辑（假设达到已获证实 → 同步点亮对应事实节点）；可信度仪表盘为 M2+ 扩展。

#### 阶段化演进规则

随着调查推进，达到「已获证实」等级的假设会同步点亮对应事实节点，逐步拼出完整真相：
- 第一阶段（现场勘查后）：点亮死者身份、死亡地点、初步死亡时间
- 第二阶段（尸检完成后）：点亮死因、关键作案特征
- 第三阶段（证人询问后）：点亮凶手外貌特征、作案工具初步判断
- 第四阶段（真相大白后）：点亮凶手身份、作案动机、完整作案过程

#### 可信度仪表盘（M2+）

> 🚧 **M2+ 扩展**：MVP 阶段不实现可信度仪表盘。

以进度条形式展示各核心结论的支持度排名，直观呈现当前推理的倾向性，包含凶手身份、作案动机、作案手法三类核心维度。

## 三、扩展进阶层模块

> 🚧 **M2+ 扩展层**：以下全部模块为 M2+ 扩展功能，MVP（血字的研究）阶段不实现。扩展模块随剧情推进逐步解锁，增加推理深度与策略性，不影响核心玩法循环。

扩展模块随剧情推进逐步解锁，增加推理深度与策略性，不影响核心玩法循环。

### 3.1 可信度评估体系
- **解锁时机**：第一章现场勘查完成后解锁
- **计算规则**：最终可信度 = 基础可信度 × 场景修正系数
  - 基础可信度按线索属性标签分为5档：直接物证0.95、专业检验0.90、无利害关系目击0.70、二手传闻/利害关系人证词0.40、嫌疑人自述0.20
  - 场景修正系数：理想观察条件×1.0、一般观察条件×0.6、情绪干扰/记忆模糊×0.4
- **展示规则**：前台仅展示「高/中/低」三个等级，搭配简短说明，不暴露公式与具体数值，引导玩家优先采信高可信度证据

### 3.2 矛盾疑点册
后台常驻全量矛盾检测逻辑，前端通过三类入口实现分层触发，兼顾新手引导与自主探索。

#### 后台检测逻辑
系统后台持续预计算所有线索间、线索与假设间、假设间的矛盾关系，所有矛盾点提前完成判定，仅控制前台展示与触发时机，不做实时运算，无额外性能损耗。

#### 前端触发入口
##### 入口1：线索对比台（全难度常驻）
- 触发方式：玩家主动将两条线索拖入对比台（拖拽模式），或主动连接两条线索（自由连线模式）
- 反馈规则：存在矛盾则触发「发现疑点」奖励，并存入矛盾疑点册；无矛盾则提示「暂未发现冲突」，无任何惩罚
- 定位：纯自主探索入口，完全依靠玩家观察发现矛盾

##### 入口2：求助（全局扫描）（全难度常驻 / M2+）

> 🚧 **M2+ 扩展功能**：求助（全局扫描）功能由独立的 **HelpSystem** 提供（见《08_系统框架设计》§3.x），推理墙通过事件总线调用；MVP 阶段不实现全量扫描求助。

- **功能入口**：推理看板顶部常驻功能按钮，玩家主动点击触发
- **功能效果**：扫描当前所有活跃假设，一次性输出所有潜在矛盾、证据缺口、可深挖方向，生成简短求助报告
- **使用次数限制**：三模式统一为单案件限用3次，超出后可消耗游戏钱币购买额外次数
- **定位**：卡关时的可控求助手段，不强制推送，由玩家自主决定是否使用
- **徽章联动**：不使用任何求助/提示通关可获得「火眼金睛」专项徽章

##### 入口3：被动矛盾提示（分难度配置 / M2+）

> 🚧 **M2+ 扩展功能**：被动提示系统为 M2+ 扩展，MVP 阶段不实现自动矛盾检测与弱提示推送。

- 触发逻辑：后台检测到矛盾后，以气泡形式弱提示玩家
- 分难度规则：
  - 简单/助手模式：默认开启，有新矛盾自动推送提示，作为新手引导
  - 普通模式：默认关闭，玩家可在设置中手动开启
  - 困难模式：强制关闭，无法手动开启

#### 矛盾处理与奖励
| 处理方式 | 效果与奖励 |
|----------|------------|
| 深入调查 | 解锁隐藏线索或额外对话分支，获得「洞察之星」奖励 |
| 标记待解 | 存入疑点册，后续获取新线索时自动关联提示 |
| 暂时搁置 | 无惩罚，不影响主线通关与星级评价 |

### 3.3 案件进度轴
- **解锁时机**：第三章多场景调查开启时解锁
- **进度推进规则**：采用事件驱动的进度推进机制，仅玩家完成「场景勘查完毕」「证人询问结束」「检验流程完成」等关键调查动作时，案件进度才向前推进；玩家翻看线索、梳理推理的思考时间不影响案件进度，不打断推理节奏。
- **进度差异机制**：不同调查顺序会影响线索的信息完整度，而非直接完全失效，将时间压力转化为调查策略选择：
  - 优先勘查现场：物理线索完整度高，可获取精准测量数据
  - 优先询问证人后折返现场：物理线索出现损耗，仅能获取大致范围数据
  - 完全错过进度窗口：线索进入「已失效」状态，仅保留基础描述，无法提供精准支撑数据
- **核心交互功能**：
  1.  进度轴主视图展示案件关键进度节点，以及玩家已完成的调查动作标记
  2.  支持点击历史进度节点回溯，查看对应节点的线索状态与推理进度
  3.  展示当前待调查内容的进度提醒，仅做信息告知，不做强制倒计时
- **困难模式扩展：凶手行动线**
  > **⚠️ M2+ 扩展机制，MVP 阶段不实现。**
  > **系统归属说明**：凶手行动线逻辑由 **ActionSystem** 管理（见《08_系统框架设计》），与调查进度系统联动；本处仅说明其在推理墙进度轴上的呈现方式。
  困难模式下同步展示凶手的行动线，若玩家长期偏离主线调查，凶手会推进自身行动（如转移藏匿、再次作案、逃离本地），最终影响结局走向；该机制仅改变结局分支，不会直接导致游戏失败卡关。

## 四、全局支撑层模块
全局支撑层为推理墙提供评价、引导、存档、统计等通用能力，贯穿核心层与扩展层的全生命周期。

### 4.1 三星评价体系
三星评价体系是推理墙的核心反馈机制，从观察、推理、洞察三个独立维度对玩家的推理表现进行星级评定（各 1-3⭐）。

> **完整定义**：参见《00_标准化开发目录》§B-11.5 三星评价公式、《02_血字的研究_场景设计与流程》§4.5 推理提交与星级评价

#### 4.1.1 评价维度与判定规则

| 维度 | 图标 | 评价内容 | 核心问题 | 1⭐ | 2⭐ | 3⭐ |
|------|------|---------|---------|-----|-----|-----|
| **观察之星** | 🔍 | 证据收集完整度 | "我找到了所有线索吗？" | 缺 ≥3 条 | 缺 1-2 条 | 缺 0 条（完整） |
| **推理之星** | 🧠 | 推理正确性与逻辑性 | "我的推理正确吗？" | 部分正确（≤1/4） | 多数正确（3/4） | 全部正确（4/4） |
| **洞察之星** | 💡 | 调查方向与隐藏信息发现 | "我发现了别人没注意到的吗？" | 选错方向（绕路） | 选重要方向 | 关键→重要→一般 最优顺序 |

#### 4.1.2 评价方式
- **单条推理链总评 = 观察之星 + 推理之星 + 洞察之星**（三维独立，各 1-3⭐，单链最高 9⭐）
- **案件总评**：所有推理链星级汇总（以《血字的研究》为例，14 条推理链 × 9⭐ = 126⭐ 满星）
- **即时反馈**：每次提交推理后立即显示对应推理链的三星评价，玩家可即时知道哪些维度可提升
- **错误无惩罚**：推理错误不扣分、不衰减、不关闭推理链，仅影响推理之星的最终正确比例

#### 4.1.3 与四级验证体系的对应关系
- 四级验证体系（矛盾冲突/证据不足/倾向成立/已获证实）是后台的支持度计算机制
- 三星评价是面向玩家的最终结果反馈
- 两者关系：验证等级是过程指标，星级评价是结果指标；同一假设最终验证等级达到"已获证实"且玩家提交的推理方向正确，推理之星才能拿满

### 4.2 徽章系统
徽章系统是三星评价体系的延伸激励，通过收集型目标驱动多周目重玩。

> **完整定义**：参见《00_标准化开发目录》§B-11 评价体系相关章节、《02_血字的研究_场景设计与流程》§7 评价与徽章系统

#### 4.2.1 徽章等级
- 徽章等级由推理链星级评定决定，与难度模式无关，三种难度模式均可获得全部等级徽章
- 铜徽章（单链 3-5⭐）/ 银徽章（单链 6-8⭐）/ 金徽章（单链满星 9⭐）
- 徽章可精确计数，作为案件完成度的长期收集目标

#### 4.2.2 专项能力徽章
- 除星级徽章外，设置专项徽章（如"神枪手""博闻强识""火眼金睛""高效调查"等），奖励特定行为模式
- 专项徽章与三星评价独立计算，互不影响
- 详细清单及获取条件见各案件设计文档

### 4.3 软性引导系统
软性引导系统是推理墙卡关时的辅助机制，分为**推理墙内部提示**和**场景级软性引导**两个层次。

> **完整定义**：参见《00_标准化开发目录》§B-10.4 挫败感应对与软性引导

#### 4.3.1 推理墙内部提示（本系统范畴）

> **系统归属说明**：被动矛盾提示与求助（全局扫描）均由独立的 **HelpSystem** 统一管理（见《08_系统框架设计》），推理墙系统通过事件总线调用；以下为推理墙范畴内的触发规则说明。

| 提示类型 | 触发方式 | 说明 | 实现阶段 |
|----------|---------|------|---------|
| 被动矛盾提示 | 后台检测到新矛盾时自动弹出 | 简单模式默认开启，普通模式可手动开关，困难模式关闭 | M2+ |
| 求助（全局扫描） | 玩家主动点击触发 | 全假设一次性扫描，输出验证报告与矛盾排查；不同难度有次数限制 | M2+ |

#### 4.3.2 场景级软性引导（场景系统范畴，与推理墙联动）
- **L1 环境暗示**：场景中视觉/音效的微弱提示
- **L2 伙伴提示**：华生/赫德森太太/街头孩童等NPC对话提示
- **L3 侦探直觉**：福尔摩斯独白/思维气泡/观察笔记
- **L4 失败保护**：极端卡关时的强引导
- 12 种场景化方案（严肃刑侦/开朗活泼/神秘悬疑等）
- 提示使用次数计入洞察之星评价（提示越多，洞察之星越低）

#### 4.3.3 普通模式概率调整机制
普通模式下，系统根据玩家行为动态调整隐性提示概率：
- 5 分钟无推进 +10% 提示概率
- 3 次错误推理 +10% 提示概率
- 进度正常 -10% 提示概率
- 连续 2 场景无卡点重置为基准值

> 详细规则参见《00_标准化开发目录》§B-11.3 普通模式概率调整机制

### 4.4 推理链三段式内容结构
推理链的内容组织采用三段式结构，与推理看板的三级假设树形成对应关系：

| 三段式结构 | 对应推理墙模块 | 说明 |
|-----------|---------------|------|
| **观察事实** | 线索库 | 从场景中收集到的原始线索与证据 |
| **中间判断** | 推理看板（子假设/主假设） | 基于线索做出的推论与假设，是推理的中间层 |
| **最终结论** | 结论里程碑 | 被证实的核心事实，构成案件真相 |

- 假设树是中间判断的主要组织载体
- 玩家的推理过程就是从观察事实出发，通过中间判断，最终抵达最终结论的过程
> 详细规则参见《00_标准化开发目录》§B-10.5 推理链三段式结构

### 4.5 其他全局能力
- **存档系统**：推理墙状态（线索库/假设树/验证记录/星级评价）随案件存档同步保存与加载
- **数据统计**：记录玩家推理行为数据（错误次数、提示使用率、平均验证等级等），用于后续案件难度适配与玩家能力画像

## 五、核心玩法循环
推理墙核心玩法形成正向闭环，每一步操作均有明确反馈与目标，保障推理节奏流畅：
1.  **探索收集**：玩家在场景中完成交互，获取线索后自动收入线索库
2.  **关联推理**：通过拖拽或自由连线的方式，将线索与假设建立关联，后台自动计算支持度，前台同步更新验证等级
3.  **矛盾排查**：玩家可自主对比线索发现矛盾，或在卡关时主动使用「求助（全局扫描）」获取排查结果，解决矛盾可推进假设验证进度
4.  **验证判断**：提交假设验证，查看验证报告，根据四级验证等级决定后续行动方向（补充证据、深挖疑点、切换假设）
5.  **事实确认**：将假设推进至「已获证实」等级，对应事实节点点亮，同步更新结论里程碑
6.  **闭环结案**：所有核心事实全部确认，推理链完整闭合，完成案件侦破

## 六、难度适配体系
全难度共用同一套后台逻辑与内容配置，通过信息透明度、功能权限、规则强度的差异化配置，适配不同层级玩家需求。

| 机制模块 | 简单/助手模式 | 普通模式 | 困难模式 |
|----------|---------------|----------|----------|
| 场景线索提示 | 高亮标记所有可交互点 | 微光标记关键交互点 | 无任何提示标记 |
| 默认交互模式 | 拖拽模式，支持手动切换 | 拖拽模式，支持手动切换 | 拖拽模式，支持手动切换 |
| 可信度展示 | 显示等级+详细来源说明 | 显示高/中/低三档等级 | 不显示可信度标识，玩家自行判断 |
| 假设引导 | 自动高亮优先验证的假设方向 | 弱提示推荐调查方向 | 无引导，完全自主判断 |
| 矛盾被动提示 | 默认开启，新矛盾自动推送 | 默认关闭，支持手动开启 | 强制关闭，不可手动开启 |
| 求助（全局扫描） | 单案件限用3次，超出可钱币购买 | 单案件限用3次，超出可钱币购买 | 单案件限用3次，超出可钱币购买 |
| 验证报告内容 | 完整展示支持/反对证据+明确行动建议 | 展示核心依据+模糊方向建议 | 仅显示验证等级，无详细分析 |
| 进度时效规则 | 无线索失效，所有线索永久有效 | 线索存在信息完整度差异，不会完全失效 | 严格进度规则，线索可完全失效 |
| 凶手行动线 | 无 | 无 | 同步运行，影响结局走向 |

    - 设计规范：高度精简，以进度条+图标为主要视觉元素，不占用核心操作视野

## 九、落地执行规范
1.  **数据一致性**：所有模块共用同一套线索、假设、标签基础数据，修改基础数据需同步校验所有关联模块的逻辑正确性。
2.  **配置标准化**：所有线索、假设、矛盾点均采用统一模板配置，新增内容需严格遵循标签体系规范，保障自动计算逻辑的准确性。
3.  **性能要求**：后台矛盾检测、支持度计算均采用预计算+缓存机制，保证前端操作响应延迟低于100ms，无明显卡顿。
4.  **兼容性要求**：交互模式切换、难度切换需做到无感知过渡，当前推理进度、线索状态、假设关联完全互通，不丢失玩家数据。
