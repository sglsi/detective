## 设计理念

以原著忠实改编为基础、以线索分类与调查顺序选择为核心、以推理墙为中枢、以时间与误导机制增强沉浸的网页侦探样板关卡。从而能让玩家像福尔摩斯一样观察、像侦探一样整理证据、像调查者一样分配时间、像推理者一样犯错与修正、最终比拼破案效率与判断质量。

### 核心循环

```
观察（点/拉/放大） → 归纳（线索墙归类） → 判断（分支选择/假设） → 验证（与新证据交叉） → 修正（推翻/重建假设） → 破案（推理链完整）
```

### 设计原则

1. **原著忠实**：剧情节点不能与小说矛盾，细节可增可减但不能改变案件事实。
2. **线索数据化**：每条线索是一行数据，结构稳定可被全场景复用。
3. **观察是手段、推导是目的**：观察到的细节必须能转化为可验证的结论，否则只是氛围。
4. **不惩罚错误**：玩家可以犯错，但错误必须可见、可修正、可学习。
5. **模板可复制**：本设计稿的所有结构、字段、表单都应可被下一个案件直接套用。

## 二、场景一：教学引导（四步拆解）

> **📍 规范引用**：本节实现项目级教学场景规范（4步教学法）
> 详见 [00_标准化开发目录 v2.0](./00_标准化开发目录.md) **C-14.2《教学场景规范（场景一）》** + **C-15《场景工单》**
> 场景数据结构遵循 v2.0 C-15.1 字段（11字段）

> **场景定位**：初期教学。玩家在此场景学会"观察-归纳-推理-验证"四步法，并形成对整套玩法的肌肉记忆。
> **时间**：上午10:00 | **地点**：贝克街221B实验室/会客厅
> **核心目标**：让玩家从"看故事"切换到"找线索-做推导-找证据"模式。

### 2.1 教学四步法

| 步骤 | 名称 | 福尔摩斯给的信息 | 玩家操作 | 福尔摩斯反馈/指导 |
|---|---|---|---|---|
| **Step 1** | **引导** | "一个人的手指甲、衣袖、靴子和裤子的膝盖部分、大拇指与食指之间的趼子、表情、衬衣袖口等，不论从以上哪一点，都能清晰地显露出他的职业。" | 玩家听对话，标记"可观察"对象 | 无 |
| **Step 2** | **玩家观察**（华生） | 提示："华生身上有值得观察的细节" | 玩家在华生身上找4个部位（手腕/左臂/面色/站姿），每点一次获得一条推理碎片 | 每次点击给出对应推理（如"从手腕皮肤黑白分明来看，这不是您原来的肤色"） |
| **Step 3** | **玩家验证**（信使） | 提示："试试看你能不能像福尔摩斯一样，从一位访客身上找出他过去的痕迹" | 玩家在信使身上找1-2个部位（手背锚形文身/络腮胡子/步态），自主得出"海军陆战队军士"结论 | 福尔摩斯不直接给答案，仅在玩家给出错误结论时纠正："再仔细看看他的手。" |
| **Step 4** | **指导** | 福尔摩斯总结：①观察不是目的，**推导才是目的**；②**推导是基于证据的有限结论**——任何观察只能得出"可能"，必须靠多条证据交叉才能"确定" | 玩家在推理墙记录"华生=军医"的推理链 | 弹出教学卡片："试试用推理墙记录你刚才的发现" |

### 2.4 教学关失败处理（无惩罚）

- **未使用推理墙**：弹出引导"试试把你的发现记到推理墙"，不影响流程。
- **不扣分原则**：教学关无任何惩罚性扣分，所有反馈均为正向引导，鼓励玩家尝试。

## 四、推理墙系统

> **📍 规范引用**：本节实现项目级推理墙规范
> 详见 [00_标准化开发目录](./00_标准化开发目录.md) **B-10《推理墙系统》** + [06_推理墙运行机制](./06_推理墙运行机制.md)
> 核心架构：**三层架构+五区布局**（核心基础层/扩展进阶层/全局支撑层 + 顶部功能栏/左侧线索库/中央推理看板/右侧扩展面板/底部进度栏）
> 关键子节：B-10.1推理墙架构 + B-10.2分层设计（MVP基础态+M3+满级态）+ B-10.3错误推理保留 + B-11.5三星评价体系

### 4.1 系统定位

推理墙是玩家的"思维工作台"，所有线索、假设、推理链、验证结果都在这里汇集、组织、推翻、重建。  
它是本游戏与"小说复述式"游戏最核心的区别——玩家不是看剧情，是建推理。

> 完整系统定义见《06_推理墙运行机制》，本节仅记录**本案特异性**的配置与要求。

### 4.2 整体架构（三层+五区）

推理墙采用项目统一的**三层架构+五区布局**：

**核心基础层（MVP必做，开局常驻）：**
- 线索库（左侧区）—— 全案线索集中管理，支持按**人物/场景/物证类型/假设关联**等维度筛选
- 推理看板（中央区）—— 三级假设树（核心问题→主假设→子假设）+ 双交互模式（拖拽/自由连线）
- 四级验证体系 —— 矛盾冲突/证据不足/倾向成立/已获证实，4档验证等级
- 结论里程碑（底部进度栏）—— 案件侦破进度可视化，事实节点逐阶段点亮

**扩展进阶层（分阶段解锁）：**
- 可信度评估 —— 线索可信度5档分级，前台显示高/中/低三档
- 矛盾疑点册 —— 后台常驻检测，3类前端入口（线索对比台/求助全局扫描/被动矛盾提示）
- 案件进度轴 —— 事件驱动进度推进，线索信息完整度差异机制

**全局支撑层：**
- 三星评价体系（观察之星/推理之星/洞察之星）
- 徽章系统（铜/银/金三级 + 专项能力徽章）
- 难度适配系统（简单/普通/困难三档差异化配置）
- 存档与统计

> 注：原"5页签"（人物/场景/物证/推断/待验证问题）现为**线索库的筛选维度**，非独立页签。"待验证问题"为玩家自行记录的开放疑问列表，归入线索库扩展功能。

### 4.3 页面布局

> 标准布局规范见《06_推理墙运行机制》§七 UI布局标注图说明。本案采用标准五区布局，无特异性UI调整。

**五区布局概要：**
1. **顶部功能栏**：返回按钮、案件名称、设置、导出
2. **左侧线索库区**：全案线索列表，支持按人物/场景/物证类型/关联假设筛选
3. **中央推理看板区**：三级假设树 + 拖拽/自由连线双交互模式
4. **右侧扩展面板区**（可折叠）：可信度评估、矛盾疑点册、案件进度轴
5. **底部进度栏**：结论里程碑进度、星级评价、当前场景

    var contradict_evidence: Array      # 矛盾证据ID列表
    var missing_evidence: Array         # 缺失证据类型列表
    var timestamp: int                  # 验证时间戳

# 求助-全局扫描（高级功能）
class HelpInspectionSystem:
    var total_attempts_used: int = 0    # 已使用次数（按难度限次）
    var max_attempts: int               # 最大次数（简单=无限/普通=2/困难=3）
    var difficulty_mode: enum           # EASY / NORMAL / HARD
    
# 单次检验报告
class HelpInspectionReport:
    var id: String                      # 报告ID：HELP_INSPECT_xxxx
    var conflict_hypotheses: Array      # 矛盾假设列表
    var insufficient_hypotheses: Array  # 证据不足假设列表
    var exploration_directions: Array   # 可深挖方向列表
    var report_content: String          # 自然语言报告内容
    var timestamp: int                  # 检验时间戳
```

**与软性引导的协同**：
- 玩家主动求助后，软性引导（B-10.4）的计时器重置，避免重复提示
- 困难模式可关闭软性引导，但保留"求助"按钮（给硬核玩家掌控权）
- MVP阶段必须实现：逐条验证（基础玩法） + 轻量版全局扫描（输出矛盾假设+证据缺口，不做深度分析）

## 五、评价与徽章系统

> **📍 规范引用**：本节实现项目级评价规则与模式差异
> 详见 [00_标准化开发目录](./00_标准化开发目录.md) **C-13.6《评价规则》**（三星评价体系+徽章系统）+ **B-11《模式与难度》**（3难度固定）
> 关键约束：三星评价三维独立、错误不扣分、即时反馈——**全项目统一**，任何调整需同步更新00文档与02文档
> ⚠️ v4.0 已明确：项目**不提供**新手/专家/自定义难度，避免"选困难刷星"

### 5.1 评价原则

| 原则 | 说明 |
|---|---|
| **三维独立评价** | 观察/推理/洞察三个维度独立评星，互不干扰，玩家清楚知道哪方面可提升 |
| **错误不惩罚** | 错误推理可推翻重做，不扣星、不衰减，仅记录为"试错路径" |
| **即时反馈** | 每次提交推理后立即显示该推理链的三星评价，玩家可即时调整策略 |
| **质量重于速度** | 评价与耗时无关（时间属调查进度系统范畴），鼓励谨慎推理而非抢答 |
| **路径多样不设限** | 可按任意顺序调查，最终只看结果质量，不强制"最优路线" |

### 5.2 三星评价维度

单条推理链总评 = **观察之星 + 推理之星 + 洞察之星**（三维独立，各 1-3⭐）

| 维度 | 评价内容 | 核心问题 | 1⭐ | 2⭐ | 3⭐ |
|------|---------|---------|-----|-----|-----|
| **观察之星** 🔍 | 证据收集完整度 | "我找到了所有线索吗？" | 缺 ≥3 条线索 | 缺 1-2 条 | 全部找到（完整） |
| **推理之星** 🧠 | 推理正确性与逻辑性 | "我的推理正确吗？" | 部分正确（≤1/4） | 多数正确（≥3/4） | 全部正确（4/4） |
| **洞察之星** 💡 | 调查方向与隐藏信息发现 | "我发现了别人没注意到的吗？" | 选错方向（绕路） | 选重要方向 | 关键→重要→一般 最优顺序 |

**单条推理链最高星级**：3 维 × 3⭐ = **9⭐**
**本案总满星**：14 条推理链 × 9⭐ = **126⭐**

> 完整机制定义参见 [00_标准化开发目录 §B-11.5](./00_标准化开发目录.md) + [02_血字的研究 §4.5](./02_血字的研究_场景设计与流程.md)

### 5.3 观察之星判定规则

观察之星由线索收集完整度决定，按**缺失条数离散档位**直接映射（与02文档统一，不区分线索重要性等级）。

**判定逻辑（每条推理链独立计算）**：
| 该推理链线索收集情况 | 观察之星 |
|---|---|
| 全部线索均找到（缺失 0 条） | 3⭐ |
| 缺失 1-2 条线索 | 2⭐ |
| 缺失 ≥3 条线索 | 1⭐ |

> 线索重要性分级（关键/重要/一般/误导）属于场景设计范畴，用于引导玩家注意力和设计发现难度，不直接参与观察之星的数值计算。误导线索不影响观察之星（属推理之星判断范畴）。

### 5.4 推理之星判定与错误记录

**推理之星 = 玩家最终提交判断的正确比例评价**（每条推理链独立计算）

| 正确中间判断 / 总中间判断 | 推理之星 | 反馈示例 |
|---|---|---|
| 4/4（全部正确） | 3⭐ | "推理严密，无懈可击" |
| 3/4（大部分正确） | 2⭐ | "方向正确，细节可完善" |
| 1-2/4（部分正确） | 1⭐ | "有合理之处，但关键判断存疑" |
| 0/4（全部错误） | 0⭐（不计入） | "方向偏离，建议重新审视证据" |

**核心规则**：
- **只看最终提交**：推理之星只与"最终提交时的正确判断比例"相关，试错过程不扣星
- **错误不关闭推理链**：错误推论保留在推理墙的推理看板，推理链保持开放
- **可无限次修改**：玩家可反复推翻重做，直到全部正确
- **错误次数记录（仅展示）**：`reasoning_chain_error_count` 用于推理墙推理看板展示试错过程，不参与星级计算

### 5.5 洞察之星判定规则

> **v4.0 变更**：原"中断衰减系数"改为"调查方向选择质量"——从"数值衰减"转为"星级评定"，方向选错不产生任何扣罚，仅影响洞察之星星级

每个调查分叉点预设方向等级，用于判定洞察之星：

| 方向等级 | 含义 | 洞察之星影响 |
|---|---|---|
| **关键方向** | 直接推进主线推理的调查路径 | 基础分 3⭐ |
| **重要方向** | 补充关键证据、辅助验证的路径 | 基础分 2⭐ |
| **一般方向** | 背景信息、人物刻画类路径 | 基础分 1⭐ |
| **误导方向** | 看似相关实际偏离主线的路径 | 基础分 1⭐（与一般方向同级） |

**判定规则（每条推理链独立计算）**：
- 全程选择关键方向且按关键→重要→一般顺序调查 → **3⭐**
- 选了重要方向但未走误导/绕路 → **2⭐**
- 仍有关键方向未调查时，先选了一般/误导方向并提交判断 → **1⭐**

**数据记录**：`reasoning_chain_direction_log`（方向选择历史），仅用于洞察之星评定和玩家行为分析

### 5.6 徽章等级与结局分支

徽章是三星评价体系的延伸激励，通过收集型目标驱动多周目重玩。

**本案徽章清单**：
| 徽章 | 等级 | 获取条件 |
|---|---|---|
| 🔹 初出茅庐 | 铜徽章 | 单条推理链总评 3-5⭐（推理之星 ≥1⭐ 即可） |
| 🔸 渐入佳境 | 银徽章 | 单条推理链总评 6-8⭐ |
| 🏆 侦探本色 | 金徽章 | 单条推理链满星 9⭐ |
| 🎯 神枪手 | 专项徽章 | 某条推理链首次提交即全对（零错误） |
| 📚 博闻强识 | 专项徽章 | 收集所有线索（关键+重要+一般+误导） |
| 🔍 火眼金睛 | 专项徽章 | 不使用任何求助/提示通关 |
| ⏱️ 高效调查 | 专项徽章 | 单场景调查次数最少路径通关 |

> **徽章等级规则**：徽章等级由推理链星级评定决定，与难度模式无关。三种难度模式（简单/普通/困难）均可获得铜/银/金全部等级徽章。难度差异体现在引导程度和隐藏细节发现条件上。

**结局分支（按总星级占比）**：
| 结局档位 | 总星级占比 | 解锁内容 |
|---|---|---|
| **传奇侦探** | ≥90% | 全彩蛋+四签名伏笔全部明示+专属称号 |
| **杰出侦探** | 70%-89% | 部分彩蛋+四签名伏笔部分提示 |
| **合格侦探** | 50%-69% | 关键线索解锁+四签名预告 |
| **见习侦探** | <50% | 基础结局，可回溯重玩 |

### 5.7 徽章等级与难度关系

| 徽章等级 | 星级条件 | 获取难度 |
|---|---|---|
| 🥉 铜徽章 | 单链总评 3-5⭐ | 三难度均可获得 |
| 🥈 银徽章 | 单链总评 6-8⭐ | 三难度均可获得 |
| 🥇 金徽章 | 单链满星 9⭐ | 三难度均可获得 |

> **设计意图**：徽章等级由推理链星级评定决定，与难度模式解绑。三种难度模式均可获得铜/银/金全部等级徽章，难度差异仅体现在引导程度、线索提示强度和隐藏细节发现门槛上——从"选困难刷徽章"转为"追求更高星级"，鼓励玩家提升推理质量而非选择难度。

### 5.8 三星评价公式

```
单条推理链总评 = 观察之星 + 推理之星 + 洞察之星
（三维独立，各 1-3⭐，不连乘、不衰减）
```

```
案件总星级 = Σ（每条推理链总评）
本案满星 = 14 条推理链 × 9⭐ = 126⭐
```

| 维度 | 判定依据 | 数据来源 |
|------|---------|---------|
| 观察之星 | 线索收集完整度（关键/重要/一般） | `discovered_clues` + 线索重要性分级表 |
| 推理之星 | 最终提交判断的正确比例 | `intermediate_deductions_correct` |
| 洞察之星 | 调查方向选择质量 | `reasoning_chain_direction_log` |

### 5.9 过度探索处理

- **不扣星**：探索全部物证不影响任何星级评价。
- **专项徽章激励**：收集所有线索（含一般和误导）可获得"博闻强识"徽章。
- **不影响主线评价**：三星评价只基于推理质量和关键证据，不强制要求全收集。

  settings: object  # 音量/字幕/UI偏好
  
  # 元数据
  save_time: timestamp
  case_version: string
```

### 6.6 模板迭代规则

- **每月一次**：从已发布案件中收集玩家数据，识别模板瓶颈。
- **每3个案件一次**：评估模板是否需要结构调整（字段合并/拆分/淘汰）。
- **不破坏向下兼容**：模板字段只能新增不能删除，老案件自动兼容。
- **变更记录**：每次模板变更需写入 `维多利亚伦敦探案项目/设计文档/案件模板变更记录.md`。
