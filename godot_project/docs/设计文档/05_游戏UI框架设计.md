## 一、整体设计规范

### 1.1 视觉基调

游戏整体视觉语言为**19世纪维多利亚复古探案风格**。核心氛围关键词：泛黄做旧、烫金浮雕、暗红连线、火漆印章、写实油画、羊皮纸卷宗。

**设计哲学：沉浸优先。** 界面是维多利亚世界的窗口，不是操作面板。所有导航元素在无操作时自动隐去，让玩家看到的是一张完整的维多利亚油画——只有需要交互时，UI才优雅浮现。

每个界面都像是一份从贝克街221B档案室取出的旧卷宗。界面之间通过统一的材质肌理与色彩体系保持视觉连贯性。

**视觉基准界面**：游戏内容界面（UI预览_横版_游戏内容界面.png）——所有界面的色调、材质、字体、装饰风格均以此为准。

### 1.4 图标与装饰规范

**图标风格**：复古浮雕金属风格，烫金/红棕底色+深色图样，带立体凹凸质感（如功能菜单栏中的放大镜、羽毛笔等图标）。

**装饰元素**：
- **羊皮纸肌理**：所有界面底色叠加泛黄旧纸张颗粒纹理与褶皱
- **烫金雕花边框**：顶部装饰栏、侧边栏外框使用金色浮雕花纹边框
- **红棕色雕花包裹**：游戏主界面左侧功能栏以红棕色雕花边框整体包裹
- **火漆印章**：委托信右下角红色火漆封印，作为剧情信件的标志性装饰
- **做旧毛边**：纸张、卡片、地图边缘做不规则磨损与毛边处理
- **侦探道具装饰**：放大镜（正负形效果）、怀表、旧报纸、老照片、烟斗等作为角落装饰元素

### 1.5 动效规范

| 动效类型 | 效果描述 | 时长 | 应用场景 |
|---------|---------|------|---------|
| 页面切换 | 旧纸张翻页效果+轻微颗粒噪点过渡 | 200ms | 界面间切换 |
| 按钮交互 | 烫金光泽流动效果+轻微凹凸反馈 | 100ms | 所有按钮hover/点击 |
| 推理连线 | 暗红墨水沿路径流动绘制 | 500ms/段 | 推理板连线出现 |
| 对话出现 | 打字机逐字出现+轻微墨水晕染 | 50ms/字 | 对话文本显示 |
| 线索解锁 | 微光扫过+烫金高亮+逐渐清晰 | 400ms | 新线索发现 |
| 委托信打开 | 火漆印章裂开+信纸展开动画 | 600ms | 委托信查看 |
| 导航栏展开 | 从边缘滑入+旧纸张质感渐显 | 250ms | 顶部栏/左侧栏唤出 |
| 导航栏收起 | 向边缘滑出+消融为1-2px金线 | 200ms | 顶部栏/左侧栏隐藏 |
| 底部对话栏展开 | 从屏幕底部滑入+轻微纸张展开 | 250ms | TALK对话触发 |
| 底部对话栏收起 | 向屏幕底部滑出+画面下缘干净收尾 | 200ms | 对话结束 |
| 场景暗化 | 场景亮度降至60%+轻微暗角加深 | 150ms | 对话/观察模式时 |

### 1.6 界面间导航关系

```

                │                │
                │                ├─ LOOK → 观察模式（高亮可交互元素）
                │                ├─ TALK → 对话系统（底部对话栏展开）
                │                ├─ EXAMINE → 线索详情界面
                │                ├─ THINK → 推理板界面
                │                └─ JOURNAL → 日志/案件进度

                ├─ INVENTORY → 物品界面（分类浏览+物品详情）
                ├─ OPTIONS → 选项栏界面（游戏设置）

                └─ 设置 → 游戏设置
```

## 二、统一布局架构与隐藏机制

### 2.1 三层框架+可隐藏底栏总览

游戏除启动主界面外，所有游戏内界面共享同一套框架结构：**顶部状态栏 + 左侧指令栏 + 中央内容区 + 底部对话栏**。前三层构成常驻框架，底部对话栏按需出现。

**核心设计原则：沉浸优先，导航可隐。**

```
┌────────────────────────────────────────────────────────────────────────────┐
│ ★ 第一层：顶部状态栏（可隐藏）                                              │
│ ┌───────────────────────┬──────────────────────────────────────────────────┐│
│ │ 位置标识      时间标识  │  MAP  CASEBOOK  EVIDENCE                      ││
│ │ 221B BAKER STREET     │  INVENTORY  OPTIONS              DAY5 上午10:30 ││
│ └───────────────────────┴──────────────────────────────────────────────────┘│
├──────────┬─────────────────────────────────────────────────────────────────┤
│ ★ 第二层：│ ★ 第三层：中央内容区（横版主视觉区域）                             │
│ 左侧指令栏│                                                                 │
│ （可隐藏）│  （各界面差异化内容在此区域呈现）                                   │
│          │                                                                 │
│ LOOK 观察│  游戏主界面 → 伦敦地图 + 委托信 + 道具区                           │
│ TALK 对话│  游戏内容界面 → 写实场景 + 人物 + 对话栏                           │
│ EXAMINE  │  推理板界面 → 线索关联图 + 推理笔记本                               │
│  调查    │  线索详情界面 → 物品特写 + 观察记录 + 关联                         │
│ THINK 思考│  案件进度界面 → 进度卷轴 + 章节列表                                │
│ JOURNAL  │                                                                 │
│  日志    │                                                                 │
│ SAVE 保存│                                                                 │
│ LOAD 读取│                                                                 │
│          │                                                                 │
└──────────┴─────────────────────────────────────────────────────────────────┘
│ ★ 底部对话栏（可隐藏，仅TALK/LOOK时展开）                                    │
│ SHERLOCK HOLMES: "The evidence is most intriguing..."                       │
└────────────────────────────────────────────────────────────────────────────┘

★ = 可隐藏组件，无操作5秒后自动收起，收起后仅留1-2px烫金边缘提示条
```

### 2.2 第一层：顶部状态栏（可隐藏）

**位置**：界面顶部，横向贯穿全宽  
**风格**：烫金浮雕字体 + 做旧羊皮纸底色，与整体复古卷宗风格统一  
**内容分三区**：

| 区域 | 位置 | 内容 | 说明 |
|------|------|------|------|
| 位置标识 | 左侧 | 「221B BAKER STREET」 | 当前角色所在地点，随场景切换动态更新 |
| 功能按钮 | 中部偏右 | MAP / CASEBOOK / EVIDENCE / INVENTORY / OPTIONS | 五个全局功能入口，烫金浮雕图标+文字，有未读内容时右上角显示红色感叹号提示 |
| 时间标识 | 右侧 | 「DAY 5 上午10:30」 | 游戏内日期与时间，随剧情推进自动更新 |

**隐藏机制**：

| 状态 | 表现 | 触发条件 |
|------|------|---------|
| 展开状态 | 完整显示位置标识+功能按钮+时间标识，做旧羊皮纸底色+烫金雕花边框 | 用户主动唤出 / 刚执行完操作（5秒内） |
| 收起状态 | 仅在屏幕顶缘保留1-2px烫金金线，其余全部隐去 | 无操作5秒后自动收起 |
| 唤出方式（PC） | 鼠标悬停至屏幕顶部边缘（距顶缘20px内）→ 自动展开；快捷键Esc | 立即唤出，250ms滑入动画 |
| 唤出方式（手机） | 点击顶部1-2px烫金金线 → 展开 | 点击即唤出，250ms滑入动画 |

**交互特性**：
- 展开后无操作5秒自动收起
- 用户正在操作功能按钮时（hover/点击中）不触发收起
- 手动收起：PC端再次按Esc / 点击金线提示条（展开状态下金线变为「▲」收起按钮）

### 2.3 第二层：左侧指令栏（可隐藏）

**位置**：界面左侧，垂直排列  
**风格**：烫金浮雕字体 + 复古图标，游戏主界面为红棕色雕花边框包裹，游戏内容界面为纯烫金栏  
**内容**：7个动作指令，覆盖全部游戏交互

| 指令 | 英文 | 中文 | 功能 |
|------|------|------|------|
| 观察 | LOOK | 观察 | 进入观察模式，场景中可交互元素以烫金高亮提示 |
| 对话 | TALK | 对话 | 底部对话栏展开，与在场NPC交谈 |
| 调查 | EXAMINE | 调查 | 对特定物品/线索近距离检查，跳转至线索详情 |
| 思考 | THINK | 思考 | 进入推理板界面，查看当前推理进度 |
| 日志 | JOURNAL | 日志 | 打开日志，查看案件进度与历史记录 |
| 保存 | SAVE | 保存 | 存档当前游戏进度 |
| 读取 | LOAD | 读取 | 读取已保存的游戏存档 |

**隐藏机制**：

| 状态 | 表现 | 触发条件 |
|------|------|---------|
| 展开状态 | 完整显示图标+文字标签，红棕色/烫金雕花边框包裹 | 用户主动唤出 / 刚执行完操作（5秒内） |
| 收起状态 | 仅在屏幕左缘保留1-2px烫金金线，其余全部隐去 | 无操作5秒后自动收起 |
| 唤出方式（PC） | 鼠标悬停至屏幕左侧边缘（距左缘20px内）→ 自动展开；快捷键Tab | 立即唤出，250ms滑入动画 |
| 唤出方式（手机） | 点击左侧1-2px烫金金线 → 展开 | 点击即唤出，250ms滑入动画 |

**交互特性**：
- 展开后无操作5秒自动收起
- 用户正在操作指令按钮时（hover/点击中）不触发收起
- 当前可用指令正常显示，不可用指令灰化处理（`#6B5B4E`灰棕+透明度50%）
- 选中指令时烫金光泽流动反馈
- 手动收起：PC端再次按Tab / 点击金线提示条

**游戏主界面的左侧栏差异**：
- 游戏主界面因以地图浏览为核心，左侧栏内容为导航型——「侦探笔记」「线索分析」「人物关系」「案情推理」「荣誉成就」，红棕色雕花边框包裹
- 「更换装扮」功能已移至设置（OPTIONS）面板中，「荣誉成就」替代原位置
- 进入具体场景后，左侧栏切换为动作指令型（LOOK/TALK/EXAMINE等），两套左侧栏在不同界面模式下各司其职

### 2.4 第三层：中央内容区

**位置**：顶部状态栏之下、左侧指令栏之右，占据屏幕主体  
**特性**：这是唯一随界面类型变化的区域，其余框架层在所有界面中保持固定（但可隐藏）

| 界面 | 中央内容区布局 | 内容说明 |
|------|--------------|---------|
| 游戏主界面 | 地图主体+底部双区 | 1891年伦敦立体浮雕地图居中，左下角道具区，右下角委托信区 |
| 游戏内容界面 | 场景画面+人物 | 写实油画质感19世纪伦敦场景占满中央，前景人物半身像 |
| 推理板界面 | 左右分栏 | 左65%：线索关联图，右35%：推理笔记本 |
| 线索详情界面 | 左右分栏 | 左50%：线索物品特写，右50%：观察记录+线索关联 |
| 物品界面 | 三栏布局 | 左15%：分类导航，中45%：物品陈列，右40%：详情弹窗 |
| 选项栏界面 | 三栏布局 | 左15%：指令栏，中50%：选项设置面板，右35%：NPC介绍框 |
| 案件进度界面 | 居中卷轴 | 案件进度条+章节列表 |

### 2.5 底部对话栏（可隐藏，按需展开）

**位置**：屏幕底部，横向通栏  
**风格**：做旧羊皮纸肌理底色，烫金浮雕说话人名称，复古衬线体文本  
**出现条件**：仅在TALK对话、LOOK观察反馈、EXAMINE近距离描述时自动展开  
**消失条件**：对话/观察结束后自动收起

**隐藏机制**：

| 状态 | 表现 | 触发条件 |
|------|------|---------|
| 隐藏状态 | 完全不可见，场景画面延伸到屏幕底缘，零UI遮挡 | 非对话状态 / 对话结束后 |
| 展开状态 | 底部1/4区域覆盖式展开，场景画面暗化至60% | TALK / LOOK / EXAMINE触发 |
| 展开动画 | 从屏幕底部滑入+轻微纸张展开质感 | 250ms |
| 收起动画 | 向屏幕底部滑出，画面下缘干净收尾 | 200ms |

**关键设计**：
- 底部对话栏底色为做旧羊皮纸肌理，左侧标注说话人名称（烫金浮雕字体）
- 双语展示：英文原文（衬线体）+ 中文翻译，增强维多利亚时代沉浸感
- 玩家回复选项以卡片形式排列在对话文字下方，hover时烫金高亮
- 对话历史通过JOURNAL日志查看回顾，不在场景画面中叠加历史

### 2.6 隐藏机制总则

**自动收起规则**：
- 所有可隐藏组件（顶部栏、左侧栏）在**无操作5秒后**自动收起
- "无操作"定义：鼠标不在组件区域内、无键盘快捷键操作、无触屏点击
- "操作中"豁免：鼠标hover在组件区域内、正在拖动滑块、正在输入文字时，暂停收起计时
- 底部对话栏不适用5秒规则，而是按对话逻辑收起（对话结束即收起）

**PC端交互**：

| 组件 | 唤出方式 | 收起方式 |
|------|---------|---------|
| 顶部状态栏 | 鼠标悬停至顶缘20px内 / 按Esc | 5秒无操作自动收起 / 按Esc / 点击「▲」收起按钮 |
| 左侧指令栏 | 鼠标悬停至左缘20px内 / 按Tab | 5秒无操作自动收起 / 按Tab / 点击金线 |

### 2.7 框架切换逻辑

```
启动主界面（无统一框架，独立全屏设计）
    │
    ▼ 进入游戏
游戏主界面（统一框架 · 地图模式）
    │ 左侧栏 = 导航型（侦探笔记/线索分析/人物关系/案情推理/荣誉成就）
    │ 中央 = 伦敦地图 + 委托信 + 道具区
    │ 顶部栏/左侧栏均可隐藏
    │
    ▼ 点击地图地点
游戏内容界面（统一框架 · 场景模式）
    │ 左侧栏 = 指令型（LOOK/TALK/EXAMINE/THINK/JOURNAL/SAVE/LOAD）
    │ 中央 = 场景画面
    │ 底部 = 对话栏（按需展开）
    │ 顶部栏/左侧栏/底部栏均可隐藏
    │
    ├── EXAMINE → 线索详情界面（统一框架 · 场景模式延续）
    ├── THINK → 推理板界面（统一框架 · 推理模式）
    ├── INVENTORY → 物品界面（统一框架 · 物品模式，中央三栏布局）
    ├── OPTIONS → 选项栏界面（统一框架 · 设置模式，中央三栏布局）
    └── JOURNAL → 案件进度界面（统一框架 · 日志模式）
```

**框架切换规则**：
- 从启动主界面进入游戏主界面时，统一框架出现（顶部状态栏+左侧栏+中央内容区）
- 在游戏主界面（地图模式）与游戏内容界面（场景模式）之间切换时，左侧栏内容同步切换（导航型↔指令型），顶部状态栏保持不变
- 从游戏内容界面进入推理板/线索详情/案件进度时，统一框架保持不变，仅中央内容区替换
- 任何界面均可通过顶部MAP按钮返回地图模式，通过顶部CASEBOOK/EVIDENCE按钮跳转对应功能
- 界面切换时，所有导航栏先收起，新界面加载完成后按用户偏好状态展开

## 七、场景设计详述

场景是游戏内容界面中央区域的核心视觉载体，每个场景对应地图上的一个可探索地点。玩家点击地图地点后，中央内容区切换为该地点的场景画面，场景内包含可交互物品、NPC、观察热点等元素，通过左侧指令栏（LOOK/TALK/EXAMINE等）进行交互。

### 7.1 场景通用数据结构

```
Scene {
  id: string,                       // 场景唯一ID
  name: string,                     // 场景名称（中文）
  nameEn: string,                   // 场景名称（英文）
  locationId: string,               // 对应地图地点ID
  backgroundImage: sprite,          // 场景背景图资源
  views: SceneView[],               // 场景视角列表（支持多视角切换）
  interactables: Interactable[],    // 可交互物体列表
  npcs: NPC[],                      // 场景内NPC列表
  ambientSound: string,             // 环境音效ID
  lighting: enum,                   // 光照类型：DAYLIGHT/TWILIGHT/LAMPLIGHT/CANDLELIGHT
  timeVariant: boolean,             // 是否支持昼夜变化
}

SceneView {
  id: string,                       // 视角唯一ID
  label: string,                    // 视角中文名称
  labelEn: string,                  // 视角英文名称
  image: sprite,                    // 视角图片资源
  hotspots: Hotspot[],              // 该视角下的可交互热点区域
  navLinks: NavLink[],              // 该视角可跳转的其他视角
}

Hotspot {
  id: string,                       // 热点ID
  shape: enum,                      // 形状：RECT/POLYGON/CIRCLE
  coords: number[],                 // 热点区域坐标（相对图片的百分比坐标）
  interactableId: string,           // 关联的可交互物体ID
  defaultCursor: enum,              // 默认鼠标样式：DEFAULT/LOOK/EXAMINE/TALK
  hoverEffect: enum,                // 悬停效果：GLOW_HIGHLIGHT/PULSE/OUTLINE
  requiredItem: string,             // 触发交互所需的物品ID（可为空）
}

NavLink {
  targetViewId: string,             // 目标视角ID
  direction: enum,                  // 导航方向：LEFT/RIGHT/FORWARD/BACK
  arrowPosition: number[],          // 导航箭头在画面中的位置（百分比坐标）
}

Interactable {
  id: string,                       // 物体ID
  name: string,                     // 名称
  type: enum,                       // 类型：PROP/CLUE/DOOR/CONTAINER/DEVICE
  description: string,              // LOOK时显示的描述文字
  examineResult: string,            // EXAMINE时显示的详细观察文字
  linkedClueId: string,             // 关联线索ID（可为空）
  isLocked: boolean,                // 是否需要特定物品/条件才能交互
  unlockCondition: string,          // 解锁条件描述
  state: enum,                      // 物体状态：DEFAULT/EXAMINED/USED/BROKEN
  stateTriggers: StateTrigger[],    // 状态变更触发器
}
```

### 7.2 场景视角切换规则

- 每个场景支持1-4个视角，通过画面左右两侧的导航箭头（◀ ▶）切换
- 切换时采用平移过渡动画（300ms），模拟人物转头视角
- 视角名称标签显示在画面底部中央（浅棕底+复古字体），切换时淡入淡出
- 部分场景仅有一个视角时，隐藏导航箭头
