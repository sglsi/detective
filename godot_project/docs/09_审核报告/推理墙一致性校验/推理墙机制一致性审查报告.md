### 🔴 H-3：三级反馈与四级验证的映射关系错误

| 项目 | 内容 |
|------|------|
| **涉及文档** | 02_血字的研究_场景设计与流程.md |
| **位置** | §5.2 三级反馈机制 |
| **当前描述** | L1 正确且证据完整 → 已获证实（86-100%）<br>**L2 正确但证据不足 → 证据不足 / 倾向成立（31-85%）**<br>L3 错误或矛盾 → 矛盾冲突（0-30%）<br>即：将「倾向成立」归入"证据不足"类，引导玩家"方向正确但需继续深挖" |
| **基准描述（06 §2.3）** | 「前台呈现简化：面向玩家的场景级反馈可简化为三级（正确/存疑/错误），对应关系为：**已获证实+倾向成立→正确（绿色对号）、证据不足→存疑（黄色问号）、矛盾冲突→错误（红色叉号）**」<br>即：「倾向成立」应归入"正确"类，是正向反馈 |
| **修改建议** | 1. 将L2范围调整为"证据不足（31-60%）"<br>2. 「倾向成立（61-85%）」应归入L1"正确"类，或单独列为L1的子层级<br>3. 反馈文案需对应调整：倾向成立应给予正向肯定（"证据倾向支持该假设"），而非"证据不足"的提示<br>4. 验证触发方式（§5.2中验证提示逻辑）需同步修正 |
| **影响说明** | 映射关系直接影响玩家对假设成立度的判断，将"倾向成立"归入"不足"会误导玩家、削弱成就感 |

## 六、审查方法说明

1. **基准文档**：以《06_推理墙运行机制.md》为唯一标准，所有其他文档需向其对齐
