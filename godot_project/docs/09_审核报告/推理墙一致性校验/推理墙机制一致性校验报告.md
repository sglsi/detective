## 一、基准文档核心机制要点摘要

以下为《06_推理墙运行机制.md》中定义的核心机制，作为一致性校验的基准：

### 1.1 系统架构
- **三层架构**：核心基础层（开局常驻）+ 扩展进阶层（分阶段解锁）+ 全局支撑层
- **五区布局**：顶部功能栏 + 左侧线索库区 + 中央推理核心区（推理看板）+ 右侧扩展面板（可折叠）+ 底部进度栏

### 1.2 核心基础层（4模块）
| 模块 | 核心定义 |
|------|----------|
| **线索库** | 4种状态机（未发现/已收集/已验证/已失效）；3类标签体系（内容标签/属性标签/关联标签） |
| **推理看板** | 三级假设树（核心问题→主假设→子假设）；6种假设状态（待建立/活跃中/领先中/已确认/已排除）；双交互模式（拖拽/自由连线）；支持度计算公式（线索可信度×标签匹配系数） |
| **四级验证体系** | 矛盾冲突(0-30%) / 证据不足(31-60%) / 倾向成立(61-85%) / 已获证实(86-100%) |
| **结论里程碑** | 4阶段点亮（现场勘查后/尸检完成后/证人询问后/真相大白后）；可信度仪表盘（3类核心维度排名） |

### 1.3 扩展进阶层（3模块）
| 模块 | 核心定义 |
|------|----------|
| **可信度评估** | 5档基础可信度（直接物证0.95/专业检验0.90/无利害目击0.70/二手传闻0.40/嫌疑人自述0.20）×场景修正系数；前台显示高/中/低三档 |
| **矛盾疑点册** | 后台常驻全量检测；前端3类入口（线索对比台/福尔摩斯检验/被动矛盾提示）；无惩罚原则 |
| **案件进度轴** | 事件驱动进度推进；线索信息完整度差异机制；困难模式凶手行动线（M2+扩展） |

### 1.4 难度适配体系（3档×9维度）
简单/助手模式、普通模式、困难模式，在以下维度差异化：场景线索提示、默认交互模式、可信度展示、假设引导、矛盾被动提示、福尔摩斯检验次数、验证报告内容、进度时效规则、凶手行动线。

### 1.5 其他已定义机制
- 自由猜想功能
- 推理错误处理：不扣分、不关闭推理链、仅作过程记录
- 成就徽章系统：P2高阶扩展功能，仅提及名称，无具体定义

### 1.6 基准文档中**未明确定义**的机制
> 以下机制在其他文档中有详细定义，但基准文档中缺失或仅一笔带过：
- 三星评价体系（观察之星/推理之星/洞察之星）
- 徽章系统的具体分类与获取规则
- 线索5类重要性划分（关键/重要/一般/其他/误导）
- 软性引导4层次+12种场景化方案
- 普通模式概率调整机制
- 推理链三段式内容结构（观察事实→中间判断→最终结论）
- 5维度评分系统

## 五、整体评估

### 5.1 修改量评估

- **06文档（基准文档）**：需要补充大量内容（三星评价、徽章系统、软性引导、线索分级等），约占现有篇幅的30-40%新增量
- **00文档**：4-5处需要修改，其中2处为结构性修改（硬规范替换、B-10.2更新）
- **03文档**：推理墙章节需要重写，是单文档修改量最大的
- **其他文档**：1-2处小修改

**总修改量**：约相当于1份中等篇幅文档的重写量

### 5.2 核心问题总结

1. **基准文档（06）不完整**：作为推理墙机制的权威定义，缺少评价体系这一核心模块。这是最大的问题。
2. **版本迭代不同步**：v4.0三星评价改版已在02文档落地，但00文档的硬规范和C-13.6评分规则未同步更新。
3. **架构升级不彻底**：06文档提出了三层架构+五区布局的新设计，但00文档仍将旧的5页签列为硬规范，03文档更是完全未更新。
4. **文档分层清晰但引用滞后**：三层文档体系（L1/L2/L3）的设计思路清晰，但上层规范更新后下层工单未同步，导致不一致。

### 5.3 建议的修改顺序

1. **先补06文档**：把缺失的评价体系、徽章系统等核心机制补充完整
2. **再调00文档**：更新硬规范和系统级定义，与06文档对齐
3. **最后更新L3文档**：03文档等工单级文档跟随L1/L2更新

> 即遵循"自上而下"的修改顺序：L2（06）→ L1（00）→ L3（03/其他），避免边改边依赖的混乱。
