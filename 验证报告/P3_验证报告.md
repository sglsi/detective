## 三、P3 交付物清单（模块 8、9）

### 3.1 存储适配层 — `backend/src/db/storage.js`（新增）

解耦业务路由与底层存储，定义统一 `StorageAdapter` 接口，两套实现按 `.env` 自动选择：

| 组件 | 说明 |
|------|------|
| `StorageAdapter` | 抽象基类，定义 Auth / Saves / Progress 全部方法签名 |
| `SQLiteStorage` | 本地开发/CI，使用 Node 内置 `node:sqlite`（`DatabaseSync`，**零外部依赖**），WAL 模式 |
| `SupabaseStorage` | 生产环境，保留既有 `@supabase/supabase-js` 逻辑（`getSupabaseAdmin()`） |
| `getStorage()` 工厂 | `STORAGE_MODE` 环境变量：`local` / `supabase` / `auto`（默认）。`auto` 仅当 `SUPABASE_URL` 与 `SUPABASE_SERVICE_KEY` 均为**真实非空值**（不含 `your-project` / `your-service` 占位）时才用 Supabase，否则回退本地 SQLite |
| 密码哈希 | `crypto.scryptSync`（`hashPassword` / `verifyPassword`，带 salt，timingSafeEqual 校验） |
| 本地 JWT | `jsonwebtoken` + `JWT_SECRET`，`expiresIn: 30d`；本地模式签发/验证 token |

SQLite 表结构：`profiles`（用户名/邮箱 UNIQUE）、`game_saves`（含 `idx_saves_user_case` 索引）、`case_progress`（`UNIQUE(user_id, case_id)`）。`upsertSave` 采用「同 user+case 覆盖最新」策略（M1）。

### 3.2 路由层重写（解耦 Supabase）

| 文件 | 改动 |
|------|------|
| `src/routes/auth.js` | register/login/guest 全部改用 `getStorage()`；register 抛 `{status, message}`；login 返回 `token`（本地 JWT 或 Supabase session） |
| `src/routes/saves.js` | 使用 `getStorage()`；新增 `pickSave()` 抽取 `SAVE_FIELDS` |
| `src/routes/progress.js` | 使用 `getStorage()`；`getProgress` 缺省返回 `not_started` |
| `src/middleware/auth.js` | `authRequired` 用 `_secret()`（JWT_SECRET 或 SUPABASE_JWT_SECRET）验证 JWT；支持游客模式（`X-Guest-ID` 头 → `userId = "guest:${id}"`） |
| `src/db/migrate.js` | `npm run migrate` → Supabase 模式打印 SQL 指令；本地模式实例化 `SQLiteStorage`（自动建表 / 生成 `backend/data/local_dev.db`） |
| `.env` | `STORAGE_MODE=local`；Supabase 变量注释占位；`JWT_SECRET` / `JWT_EXPIRES_IN=30d` / `GUEST_EXPIRE_HOURS=24` |

### 3.3 `.tres` 解析 / 序列化器 — `backend/src/tres/parser.js`（重写）

| 能力 | 说明 |
|------|------|
| `parseTres(content)` | 返回 `{ meta, extResources, resource, nodes }`。**关键修复**：现可解析 `[resource]` 块（含 `scene_id` / `*_start_node` / `knowledge_domains` / `milestone_name` / `score_*` / `badge_check` / `completion_event` / `nodes=[SubResource(...)]`） |
| `serializeTres(data)` | 输出可被 Godot 4 直接加载的完整 `.tres`：**头部 → ext_resource(×2) → sub_resource(×N) → `[resource]` 块**（含 `script` + 场景元数据 + `nodes = [SubResource("node_0")...]`） |
| 字段类型表 | `NODE_TYPES` / `RESOURCE_TYPES`（string / string_array / int / float / bool），规范输出顺序，未知字段原样回写，**往返无损** |
| 通用数组解析 | 支持 `["a","b"]` 与 `[SubResource("x"), SubResource("y")]`；`parseArray` 尊重嵌套括号与引号 |

> ⚠️ 历史版本 `serializeTres` **从未输出 `[resource]` 块**，导致编辑器保存产物丢失全部场景级元数据、`nodes` 引用数组缺失，Godot 无法加载（即上一阶段 scene_04 冒烟失败的真正根因）。本次彻底修复。

### 3.4 编辑器后端 API — `backend/src/routes/editor.js`

| 端点 | 功能 |
|------|------|
| `GET /api/editor/files` | 列出对话目录所有 `.tres`（含大小/mtime）；`safeJoin` 防目录穿越 |
| `GET /api/editor/file?name=` | 解析 + 统计（node_count / speakers / step_entries / verify_branches）；返回 `resource` 块 |
| `POST /api/editor/file` | 备份原文件 → `serializeTres` 写回；缺失 `resource` 时自动从原文件回读以保留场景元数据 |
| `GET /api/editor/validate?name=` | BFS 可达性 + 悬空 `next_nodes` 引用检测 |
| `src/server.js` | 挂载 `app.use('/api/editor', editorRoutes)` |

### 3.5 编辑器前端 — `web_prototype/editor.html`（新增）

三栏布局，维多利亚暗金美术风格：

| 区域 | 能力 |
|------|------|
| 左侧文件列表 | 列出全部 `.tres`，点击加载 |
| 中间节点树 | 展示 `node_id` + `speaker` + 文本摘要，标注「步入口」「验证分支」徽章 |
| 右侧属性表单 | 编辑 12 个节点字段（node_id / speaker / mood / text / trigger / difficulty / verify / step / is_step_entry / next_nodes / stage_direction / note_text） |
| 操作 | `💾 保存文件`（携带 `resource` 元数据回传）、`🔍 校验结构`（调用 `/validate`） |

## 四、校验中暴露并修复的真实缺陷

### 4.1 本次新发现并修复

| # | 缺陷 | 位置 | 根因 | 修复 |
|---|------|------|------|------|
| 1 | **编辑器保存产物 Godot 无法加载**（`scene_04` 冒烟失败真正根因） | `parser.js: serializeTres` | 序列化器**从未输出 `[resource]` 块**，丢失场景元数据与 `nodes=[SubResource(...)]` 引用数组 | 重写 `parseTres` / `serializeTres`，完整解析并序列化 `[resource]` 块；恢复 `[gd_resource ... load_steps=N+2]` 头部计算 |
| 2 | `parseTres` 丢失 `resource` 块全部字段 | `parser.js: parseTres` | 仅处理 `sub_resource`，忽略 `[resource]` 段 | 增加 `[resource]` 块解析（`script` / 标量 / `string_array` / `SubResource` 数组） |
| 3 | 前端保存未携带 `resource`，导致场景元数据被清空 | `editor.html` + `editor.js` | POST body 仅含 `nodes/meta/extResources` | `editor.js` 增加 `resource` 透传与回读兜底；前端 `loadFile`/`saveFile` 携带 `currentResource` |
| 4 | 后端 API 测试第二次运行必现 2 失败（register 409 / login 401） | `tests/api_test.js` | 测试 `username: '测试侦探'` 为**常量**，违反 `profiles.username` UNIQUE 约束；上一轮已落库 → 本轮 409 级联 401 | 测试 `username` 改为 `测试侦探_${Date.now()}`（幂等）；并清空污染 DB 重新跑出 20/20 |

### 4.2 历史遗留（在 P3 校验中确认不影响）

| # | 事项 | 结论 |
|---|------|------|
| 5 | scene_04 冒烟「无法加载资源」 | 上一阶段误判为序列化风格差异；本次定位为 **`[resource]` 块缺失（缺陷 #1）**，已修复并由 Godot 加载校验闭环 |
