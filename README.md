# 维多利亚伦敦探案项目

基于1864年斯坦福图书馆版伦敦地图的福尔摩斯主题推理探案游戏。

---

## 项目核心理念

> 玩家不需要一张完整的1864年伦敦地图，他们需要的是一张"随着探案逐步揭开的历史面纱"。

地图不是一个"工具"，而是一个"收藏品"。玩家每破一个案子，地图就丰富一点。玩到最后，玩家手上有一张自己探索出来的、独一无二的维多利亚伦敦地图。

## 核心特色

### 1. 真实的历史地理
- 所有地点基于1864年斯坦福图书馆版伦敦地图
- 著名地点使用真实19世纪历史照片
- 每个地点都有详细的历史背景介绍

### 2. 渐进式地图解锁
- 地图不是一次性全开，而是随着案件进度逐步解锁
- 每个地点有三种状态：隐藏/已标记/已探索
- 玩家最终获得一张完整的、自己探索出来的维多利亚伦敦地图

### 3. 沉浸式探案体验
- 福尔摩斯式的线索拼接推理系统
- 丰富的NPC对话和证词
- 完整的证据链和逻辑推理过程

## 设计原则

### 最重要的原则
> **不要让玩家"找路"，让玩家"破案"**

- ❌ 不要让玩家在地图上瞎找
- ✅ 明确告诉玩家下一步该去哪、去了干什么
- ✅ 地图是服务于游戏体验的，不是游戏的目的本身
- ✅ 每个案件只加载关联瓦片的最小包围矩形，不预规划全图

### 推理试错原则（2026-06-13 确立）
> **侦破的过程，本身就是一个试错的过程，也是一个认知不断变化的过程。**

- **推理错误不扣分**：错误推论不会直接扣减玩家得分
- **推理错误不导致推理链失效**：错误推论不会让推理链判定为"失败"，推理链保持开放、可继续推进
- **推理结论只是过程记录**：玩家提交的推论（无论对错）都记录在推理墙，允许随时返回修改、删除或重新提交
- **得分系数反映试错代价**：错误通过得分系数衰减间接影响最终得分（首次正确×1.0，第一次错后正确×0.8，第二次错后正确×0.6，以此类推）

这一原则体现了福尔摩斯式探案的真实体验——真相往往是在反复假设、验证、修正中逐步浮现的。

## 技术栈

- **游戏引擎**：Godot 4.x（GDScript + Compatibility渲染器，支持Web导出+原生移动端导出）
- **地图系统**：TileMapLayer（案件驱动式瓦片加载，详见07_地图系统迁移方案）
- **数据存储**：游客临时缓存 + 注册用户云端API同步
- **后端**：统一后端API（Web端和移动端共用，保持数据一致性）
- **发布策略**：第一阶段网页游戏（打开即玩，游客+注册）→ 第二阶段手机应用端
